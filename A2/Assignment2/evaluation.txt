WHAT WORKS
- Reads every file listed in data/data_source.txt from the data directory.
  Column order differs between the supplied files, so columns are located by
  header name rather than by position.
- Blank lines at the end of a data file, in the middle of a file, and inside
  data_source.txt are skipped. Verified: test_blanklines.csv and the original
  file produce byte-identical output and both load 90 records.
- A row with one offline sensor keeps its other readings; each reading has its
  own valid flag.
- Values of -999, used in the data as a sensor-offline marker, are rejected for
  wind speed and solar radiation because neither can be negative. Without this,
  May 2007 reported an average of 6.7 km/h with a standard deviation of 144.6.
  It now reports 12.5(8.3, 6.9), against a median of 14.4.
- Option 1: average wind speed in km/h and sample standard deviation for a
  given month and year. Prints No Data when absent.
- Option 2: average ambient temperature and sample standard deviation for every
  month of a given year.
- Option 3: sample Pearson Correlation Coefficient for a chosen month across
  all loaded years, printing S_T, S_R and T_R to two decimal places.
- Option 4: writes WindTempSolar.csv with average(stdev, mad) for wind and
  temperature and total solar radiation in kWh/m2, per month. Months with no
  data are omitted; a year with no data prints the year then No Data.
- Option 5 exits.
- The template BST from Lab 10 exercise d is used unmodified. All traversals
  are driven by raw function pointers.
- std::map holds one BST per year.
- No std::vector, std::function, lambda functions or range for loops appear
  anywhere in the submission. The only linear container is the Vector class
  from the labs.

DATA STRUCTURE
- std::map<int, BinarySearchTree<WeatherRecord> >, keyed by year.
- Rationale, pros, cons and rejected alternatives are in rationale.txt.

ISSUES FOUND AND RESOLVED
- Records on the same day were being lost. WeatherRecord::operator< compared
  only the date, so the tree treated the 144 readings of a day as duplicates.
  Fixed by comparing date and then time.
- Inserting records in file order built a tree that was one long right hand
  branch. At 52,560 records this exhausted the call stack and the program
  crashed. Fixed by inserting the middle element of the sorted range first and
  repeating on each half. The BST class was not modified. Measured on a 1 MB
  stack: file order terminated with a stack overflow, median first insertion
  completed in about 0.3 seconds.
- Two files can cover the same year, so each year's records are merge sorted
  before insertion rather than assumed to be in order.

A NOTED CONSEQUENCE OF USING THE TREE
- The BST rejects a duplicate key, so records sharing a date and time are
  stored once. February 2016 appears in both MetData_Mar01-2015-Mar01-2016 and
  Metdata-Jan-Dec2016. Across all twelve files, 508,733 records are read and
  499,987 stored; 8,746 duplicates are discarded. A container without unique
  keys would have double counted those readings.
