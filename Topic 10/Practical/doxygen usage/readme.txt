ModelFile.h is the interface file.
Doxygen comments are written in .h files.
Copy/paste the comments from here and adapt/change to cater for your own .h file contents.
Check that the comments are indented left so code stands proud and easily spotted. Don't bury code in documentation.

Implementation files are the .cpp (or .c) files. Same approach as in ict159.

Doxyfile is the configuration file for Doxygen. This is the tool that extracts the documentation from the .h file into html files. You can configure doxygen from scratch every time or use this file. File/Open and pick the Doxyfile. Doxygen will get configured will all the past settings. To change for new project, just change the project name and path to source directory in doxygen and resave this file.
